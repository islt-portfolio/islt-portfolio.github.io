Accessibility Statement

Commitment to Accessibility

This portfolio is committed to ensuring digital accessibility for people with disabilities. I am continually working to improve the user experience for all visitors and apply relevant accessibility standards.

Conformance Status

This website strives to conform to Web Content Accessibility Guidelines (WCAG) 2.1 Level AA standards. These guidelines explain how to make web content more accessible to people with disabilities.

Accessibility Features

This site includes the following accessibility features:

  • Semantic HTML markup for proper document structure
  • Skip navigation link for keyboard users
  • Descriptive alt text for images
  • Adequate color contrast ratios
  • Responsive design that works across devices
  • Keyboard-navigable interactive elements
  • Closed caption and fall-back option for video.
  • Clear and consistent navigation

Compatibility

This website has been tested for compatibility with:

  • Current versions of Chrome, Firefox, Safari, and Edge browsers
  • Windows and Mac operating systems
  • Mobile devices (Android-simulators and Apple iOS)
  • Screen readers: VoiceOver compatibility

Updates in Progress

The updates below are suggestions for accessibility improvement submitted via user testing. These items highlight areas where interactive elements, navigation structures, and image handling can be refined to better meet WCAG 2.1 guidelines and provide an equitable experience for all users. Strikethroughs are completed updates.

  • Main Navigation: Fix the hidden “show/hide” element in the main navigation (likely from the skip link) so it no longer appears to screen readers unexpectedly.
  • Main Navigation: Ensure main navigation links for screen readers do not all read as “visited” and add a clear current-page indicator.
  • Home Page: Add concise descriptive alt text for the home-page headshot.
  • Program of Study: Correct in-page anchor links so they consistently jump to the correct section when using a screen reader.
  • Goals and Objectives: Resolve keyboard navigation issues, ensuring all elements are reachable and usable with keyboard alone.
  • Goals and Objectives: Revise the navigation, which currently confuses VoiceOver by reading collapsed content and unlabeled checkboxes; add meaningful instructions or replace it with a simpler table-of-contents structure.
  • Footer: Mark decorative footer icons (mail and LinkedIn) as non-interactive so screen readers announce only the link text.

Technical Specifications

This website relies on the following technologies for accessibility:

  • HTML5
  • CSS3
  • Web browsers with CSS and HTML5 support
